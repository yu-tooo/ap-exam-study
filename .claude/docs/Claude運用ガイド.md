# Claude運用ガイド

Claude Codeの拡張機能（Skills / Custom Commands / Subagents）の違いと、本プロジェクト（応用情報技術者試験 学習プロジェクト）での活用方針をまとめた参考資料。

## 1. 三者の概要と違い

### 定義

- **Skills**: モデルが自律的に活用できる「能力」。`SKILL.md` を中心に、補助ファイル（テンプレート、サンプル等）を同梱できる。
- **Custom Slash Commands**: ユーザーがスラッシュで明示起動するプロンプトテンプレート。マークダウン1ファイルで完結する。
- **Subagents**: メイン会話とは独立したコンテキストで動くサブエージェント。`Agent` ツール経由で起動し、結果を1メッセージで返す。

### 起動の違い

| | Skills | Slash Commands | Subagents |
|---|---|---|---|
| ユーザー起動 | `/skill-name` で可能 | `/command-name` が基本 | 直接起動はしない |
| モデル自動起動 | **`description` を見て自動発動** | 自動発動しない | Skills/Commandsの内部から起動 |
| コンテキスト | メイン会話と共有 | メイン会話と共有 | **独立（メインを汚さない）** |

### ファイル配置

| | パス | 形式 |
|---|---|---|
| Skills | `.claude/skills/<name>/SKILL.md` | ディレクトリ（補助ファイル可） |
| Commands | `.claude/commands/<name>.md` | 単一ファイル |
| Subagents（型定義） | `.claude/agents/<name>.md` | 単一ファイル |

## 2. 使い分けの判断基準

- **「いつ動くか」を制御したい**
  - 自動で動いてほしい → **Skills**
  - ユーザーが明示的に叩いた時だけ → **Commands**
- **「コンテキストを汚したくない／並列化したい」** → **Subagents**
- **「単一ファイルで手軽に作りたい」** → **Commands**
- **「補助ファイル（テンプレ・サンプル）を同梱したい」** → **Skills**

ユーザー向けの「入口」をCommand、その奥の「処理ロジック」をSkill、Skillの中で「重い／並列化したい部分」をSubagentに任せる、という三層構造が綺麗。

## 3. Skillsの詳細機能

### `examples/` と `templates/` ディレクトリ

公式に定められた特別な名前ではなく**慣習**だが、意味的には：

- `templates/` … Claudeが埋める雛形（フォーマットの空テンプレ）
- `examples/` … 期待される出力例（理想的なサンプル）

#### Progressive Disclosure（段階的開示）

| ファイル | 読み込みタイミング |
|---|---|
| `SKILL.md` | **常にコンテキストに含まれる** |
| サブディレクトリのファイル | **SKILL.mdから参照され、必要時のみ読み込む** |

`SKILL.md` 本体は500行以下に収め、詳細はサブファイルに逃がすのが推奨。

#### 参照の書き方

`SKILL.md` 内で**マークダウンリンクとして書く**だけ。

```markdown
出題形式は [templates/四択.md](templates/四択.md) に従う。
理想的な問題例は [examples/ネットワーク_例.md](examples/ネットワーク_例.md) を参照。
```

### `argument-hint`

`SKILL.md` のフロントマターに記述する、引数のヒント表示。

```yaml
---
name: quiz
description: 応用情報技術者試験の問題を出題する
argument-hint: "[カテゴリ] [問題数]"
---
```

- 角括弧 `[...]` で引数を囲む
- 複数引数は空白区切り
- ユーザーが `/` を打った時のオートコンプリートに表示される
- 単なる**表示用ヒント**で、構文チェックはされない

### プレースホルダ

`SKILL.md` 本文で引数を参照するには：

| 記法 | 意味 |
|---|---|
| `$ARGUMENTS` | 全引数（文字列まるごと） |
| `$0`, `$1`, `$2` | 位置引数（0始まり） |

### `!` `@` 記法

- `` !`コマンド` `` … Bashコマンド実行結果を埋め込む
- `@ファイルパス` … ファイル内容を埋め込む

これらはSkillsとCommandsの両方で利用可能。

## 4. 本プロジェクトでの活用案

### Skills案（自動発動・プロジェクト固有の振る舞い）

「**この状況になったら自動的にこの手順で動いてほしい**」用途。

| スキル案 | 発動契機 | 内容 |
|---|---|---|
| `create-markdown` ✅既存 | メモ変換時 | テキスト→マークダウン化 |
| `quiz`（出題） | 「問題出して」と言った時 | カテゴリのMDから問題生成 |
| `answer-evaluator`（採点） | ユーザーが問題に解答した時 | 解答の正誤判定・解説 |
| `変換ログ更新` | マークダウン作成・編集後 | `変換ログ.md` を自動追記 |
| `重複チェック` | 新規マークダウン作成前 | 既存ファイルとの内容重複を確認 |

### Custom Commands案（明示起動のショートカット）

「**自分が今これをやりたい**」と明示的に叩く用途。ダッシュボード／統計／状態確認系が向く。

| コマンド案 | 用途 |
|---|---|
| `/今日の進捗` | 今日変更されたマークダウン一覧 |
| `/未変換` | `メモ/` 配下でまだ変換していないファイル一覧 |
| `/弱点` | 出題ログから正答率の低いカテゴリを抽出 |
| `/復習 [カテゴリ]` | 前回間違えた問題を再出題 |
| `/カテゴリ一覧` | 各カテゴリのファイル数・最終更新日を表示 |
| `/直前確認` | 試験直前用のサマリ生成（重要事項だけ抽出） |

### Subagents案（並列・独立コンテキスト）

「**メインの会話を汚さず、並列で処理したい**」用途。Skillsから内部的に呼ぶことが多い。

| サブエージェント案 | ユースケース |
|---|---|
| **メモ一括変換** | 未変換ファイルが複数あれば並列で `create-markdown` 実行 |
| **横断模試生成** | 5カテゴリから1問ずつ並列生成して総合演習を作る |
| **カバレッジ監査** | 全マークダウンを読み「試験範囲のうちまだ書かれていない論点」を洗い出す |
| **重複検出** | 新規マークダウン作成前に既存全ファイルを横断調査 |
| **問題ストック生成** | バックグラウンドで各カテゴリ20問ずつ生成しておく |

`.claude/agents/` にプロジェクト固有のエージェント型を定義可能：

- `memo-converter`（1メモを変換）
- `quiz-generator`（1カテゴリから出題）
- `coverage-auditor`（範囲漏れ調査）

## 5. 三者の協調イメージ

```
ユーザー:「総合演習やりたい」
  ↓
Command: /総合演習 5
  ↓
Skill: quiz が自動発動
  ↓
Subagent × 5: 各カテゴリから1問ずつ並列生成
  ↓
ユーザーに5問提示
  ↓
ユーザー解答
  ↓
Skill: answer-evaluator が自動発動して採点
  ↓
Skill: 変換ログ更新（出題ログを記録）
```

## 6. 出題スキルの実装例

参考までに、`quiz` スキルを作る場合の構造例：

```
.claude/skills/quiz/
├── SKILL.md                    # 出題ルール本体（常時ロード）
├── templates/
│   ├── 四択.md                 # 四択問題の雛形
│   └── 記述.md                 # 記述問題の雛形
└── examples/
    ├── ネットワーク_例.md       # 良い問題のサンプル
    └── データベース_例.md
```

`SKILL.md` の例：

```yaml
---
name: quiz
description: ユーザーが応用情報技術者試験の問題出題を依頼した時に使用する。指定されたカテゴリのマークダウンファイルから問題を生成する。
argument-hint: "[カテゴリ] [問題数]"
---

# 出題ルール

`$0` カテゴリのディレクトリ配下のマークダウンファイルから、`$1` 問を出題する。
問題数の指定がなければ1問。

出題形式は [templates/四択.md](templates/四択.md) に従う。
理想的な問題例は [examples/ネットワーク_例.md](examples/ネットワーク_例.md) を参照。
```

使用イメージ：

- `/quiz ネットワーク 3` → ネットワーク配下から3問出題
- 「ネットワークの問題を3つ出して」と自然言語でも、`description` を見てモデルが自動発動
