# Skillでスクリプトを使うメリットと注意点

Claude Code の Skills では、SKILL.md と一緒に Python / Bash 等のスクリプトを同梱し、エージェントから実行させることができる。本ドキュメントは、その仕組みのメリットと注意点を公式ドキュメントから抜粋し、本プロジェクト（応用情報技術者試験 学習用プロジェクト）の文脈で整理したものである。

## メリット

### 1. コンテキスト効率（トークン節約）

スクリプトは「読み込まれる」のではなく「実行される」だけなので、その本文はエージェントのコンテキストを消費しない。複雑なロジックを SKILL.md 内に書き下すと、Skill を呼ぶたびにそのテキストがロードされてトークンを消費するが、スクリプトに切り出せば実行時の出力のみがコンテキストに乗る。

### 2. 決定論性・再現性

LLM に毎回ロジックを判断させると、確率的な揺らぎが避けられない。乱数選択・割合計算・JSON 集計のような決定論的な処理はスクリプトに任せたほうが、結果が安定する。シード値を固定すれば完全に再現可能にもできる。

### 3. 複雑な処理の実装

外部ライブラリ呼び出し、統計計算、ファイル走査、データ変換など、プロンプトでは現実的に書けない処理が実装できる。エージェントはスクリプトの**出力だけ**を受け取って、その結果に従って動けばよい。

### 4. 慣習的なディレクトリ構造

公式ドキュメントでは、Skill 配下に `scripts/`（または `script/`）ディレクトリを切ってスクリプトを置くパターンが推奨されている。SKILL.md からは `${CLAUDE_SKILL_DIR}/scripts/xxx.py` のように Skill のルートを基準にしたパスで参照できる。

## 注意点

### 1. 実行環境の前提

スクリプトを実行するランタイム（Python 3、Node.js 等）がユーザー環境にインストールされていることが前提になる。さらに外部パッケージ（`pip install` が必要なもの）に依存すると、ユーザーごとのセットアップ負荷が一気に上がる。**標準ライブラリの範囲で書くこと**が安全。

### 2. パス解決

エージェントがスクリプトを実行する際の作業ディレクトリは、必ずしもプロジェクトルートとは限らない。`${CLAUDE_SKILL_DIR}` を使うか、スクリプト側で `Path(__file__)` から相対的に解決するのが堅い。

### 3. セキュリティ

Skill にチェックインされたスクリプトは、ユーザーの環境で実行される。破壊的操作（ファイル削除、DB 更新等）を含む場合は、エージェントが自動で呼ばないよう Skill 側で `disable-model-invocation: true` を指定するか、ユーザーが明示的に Skill を呼び出したときだけ実行する設計にする。

### 4. 出力サイズ

スクリプトの標準出力はそのままエージェントのコンテキストに入る。大量データをそのまま print すると、せっかくのトークン節約効果が打ち消される。スクリプト側で必要な情報だけに絞ってから出力すること。

### 5. 自動コンパクション時の挙動

セッションが長くなりコンテキストがコンパクションされると、過去に呼び出した Skill の内容は削除されることがある。引き続きスクリプト経由のロジックを使いたい場合は、Skill を再度呼び直す必要がある。

## 本プロジェクト固有の論点

本プロジェクトは「応用情報技術者試験の学習用」であり、ソフトウェアプロジェクトではない。エージェントが扱うのは主にマークダウンと JSON ログである。現状 Skill にはまだスクリプトを同梱していないが、将来的には `quizログ.json` を入力に過去誤答問題から復習対象を決定する `review` スキル（仮称）を新設し、その内部処理をスクリプトに切り出すことを想定している。

この文脈で特に効くポイント：

- **ロジックの一貫性**：「過去誤答プールから何問選ぶか」「同一問題判定」「直近 status による除外」などは、エージェントの判断に委ねるとセッション間でブレやすい。スクリプトに固定化することで、毎回同じルールで復習対象が決まる。
- **`quizログ.json` の前処理に向く**：JSON のパース・フィルタ・サンプリングは Python 標準ライブラリで完結する。外部依存は不要。
- **責務の分界が明確になる**：SKILL.md は「何をするか（What）」を、スクリプトは「どう計算するか（How）」を担う。アルゴリズム調整時は Skill 全体ではなくスクリプトだけを差し替えればよい。

注意したいのは：

- **ランタイム要件**：本プロジェクトは元々コードを書かない前提なので、Python 等のランタイムをユーザー環境に追加で要求することになる。標準ライブラリだけで完結させる。
- **作業ディレクトリ**：スクリプトはプロジェクトルートにある `quizログ.json` を読みに行くことになるため、`Path(__file__).resolve().parents[N]` でプロジェクトルートを解決するか、SKILL.md 側で実行ディレクトリを固定する必要がある。
- **破壊的操作の禁止**：CLAUDE.md にある通り、本プロジェクトはエージェントによるシェルコマンド実行や git 操作を禁じている。スクリプトはあくまで読み取り＋計算結果の標準出力に留め、ファイル更新（`quizログ.json` への追記など）はエージェント本体が行う方針が安全。
- **`quiz` スキルとの責務分離**：`quiz` スキルは作問と解答記録のみを担い、過去誤答の復習機能は `review` スキル側に閉じ込める。`quiz` 側の SKILL.md にスクリプトや復習ロジックの記述を持ち込まない。
