# quiz（作問スキル） の概要

マークダウンファイルに記載されている内容から応用情報技術者試験の類似問題を作成しユーザーに出題する。
ユーザーの解答結果は `quizログ.json` に記録する。

## 処理フロー

1. ユーザーが作問を依頼する。
2. エージェントが `qCount` 問の問題を `source` から作成する（各問題に `id` を採番する。詳細は `フィールドの決定ルール`）。
3. エージェントが1問ずつ出題し、ユーザーが解答する。これを `qCount` 問繰り返す。**この間、エージェントは正誤・解説を出さない**（ヒントを与えないため）。
4. 全問の解答が完了したら、エージェントが各問題の正誤と解説をまとめて出力する。（`出力`を参照）
5. 解答結果を `quizログ.json` に記録する。（`結果記録ルール`を参照）

## 入力

### 項目定義

1. `source`：出題元となるマークダウンファイル、またはマークダウンが格納されているディレクトリへのパス。（必須）
2. `format`：問題の形式を指定する。出題形式の詳細は`問題形式について`を参照。
3. `qCount`：問題数。

### 入力ルール

||必須|デフォルト|ルール|例|
| ---- | ---- | ---- | ---- | ---- |
|`source`|○|-|相対パスもしくは絶対パス|`ネットワーク/OSI参照モデルとTCP_IPモデル.md`|
|`format`|×|4択|4択 もしくは 記述|記述|
|`qCount`|×|3|1以上10以下の整数|5|

### `source` の解決ルール

- `source` がマークダウンファイル（`.md`）を指している場合：そのファイルの内容を出題元とする。
- `source` がディレクトリを指している場合：配下（再帰的）のすべてのマークダウンファイルを統合した内容を出題元とする。`qCount` 問はその統合内容の中から作成する（どのトピックから出題するかはエージェントが判断する）。

### バリデーションルール

引数が入力ルールに違反していた場合は、直ちにSkillの実行を禁止し、ユーザーに違反していたルールを提示する。

例）`source`から出題元になるマークダウンが特定できなかった場合
```
出題元のマークダウンファイルが特定できませんでした。ファイルパスが間違えていないか確認してください。
```

例） `format`で存在しない形式を指定した場合
```
4択もしくは記述を指定してください。
```

例）`qCount`で11以上の数を指定した場合
```
問題数は1以上10以下の整数で指定してください。
```

## 出力

- 出題時は`問題形式について`に従って作成した問題を出力する。
- 出題中は正誤・解説を出さず、ユーザーの解答だけを受け付ける。
- 全問（`qCount` 問）の解答が完了したら、各問題の結果と解説をまとめて出力する。詳細は`解答結果のフォーマット（例）`を参照する。
- 出力順：「Q1出題 → A1入力 → Q2出題 → A2入力 → … → Q`qCount`出題 → A`qCount`入力 → 全問の結果・解説」。

### 問題形式について

出題形式は「4択」と「記述」の2種類存在する。

- 4択：テンプレートは[templates/4択.md](templates/4択.md) を用いる。
- 記述：テンプレートは[templates/記述.md](templates/記述.md) を用いる。

具体的な問題例は[examples/](examples/)を参照。 

### 正誤判定ルール

- **4択**：選択肢の記号（A〜D）または選択肢の文言が正解と一致すれば `pass`、それ以外は `fail`。
- **記述**：模範解答と**意味的に等価**であればエージェントの判断で `pass`、不足・誤りがあれば `fail`。文言の完全一致は要求しない。判定根拠は解説に簡潔に含める（例：「『信頼性のある』というキーポイントが含まれているため正解」）。

### 解答結果のフォーマット（例）

全問解答後、各問題について以下の形式で順番に出力する。問題番号は出題時の番号を引き継ぐ。

「4択」形式の場合
``` console
問題1：TCP/IPにおいて、データの信頼性確保・順序管理・再送制御を行う層はどれか。
あなたの解答：(B)トランスポート層
結果：正解
解説：{ 解説を500文字以内を目安に記述する。}
```
「記述」形式の場合
``` console
問題2：OSI参照モデルにおいて、トランスポート層の主な役割を1つ答えよ。
あなたの解答：パケットの転送経路を決定する
結果：不正解
解説：{ 解説を500文字以内を目安に記述する。}
```

## 結果記録ルール

すべての問題に対して解答が完了したら、「解答結果」の出力と同時に解答結果を記録する。

- **ファイル名**：`quizログ.json`
- **配置場所**：プロジェクト直下
- **ファイルが存在しない場合**：エージェントが `{ "sessions": [] }` で新規作成し、そこに今回のセッションを追加する。ユーザーへの通知は不要。

### フィールドの決定ルール

- `timestamp`：解答結果を記録した日時（ISO 8601 形式 `YYYY-MM-DDTHH:mm:ss`、ローカルタイム）。
- `category`：`source` のパスの第1階層ディレクトリ名（例：`ネットワーク/OSI参照モデル.md` → `"ネットワーク"`、`ストラテジ/` → `"ストラテジ"`）。
- `format`: "4択"もしくは"記述"
- `contents[].id`：問題ごとに一意な ID。新規作問時に `q_` + 8 文字の英小文字・数字（例：`q_a3f2b1c4`）でランダム採番する。将来、過去の問題を再出題する場合は、**同じ `id` を引き継いで記録する**こと（同一 `id` の最新エントリの `status` が、その問題の現状ステータスとなる）。
- `contents[].No`：そのセッション内での通し番号（1始まり）。
- `contents[].status`：正解なら `"pass"`、不正解なら `"fail"`。
- `contents[].question`：出題した問題文。
- `contents[].answer`：正解（模範解答）。
- `contents[].userAnswer`：ユーザーが実際に入力した解答。4択問題の場合は選択肢の記号ではなく、選択肢の内容を記入すること。（例：○ ER図 × ア）
