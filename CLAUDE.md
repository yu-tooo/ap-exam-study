# CLAUDE.md

すべての応答は日本語で行う。

## プロジェクト概要

このプロジェクトはユーザーがIPAの応用情報技術者試験（Applied Information Technology Engineer Examination）を学ぶためのプロジェクトである。**ソフトウェアプロジェクトではない。** - ビルド、テスト、アプリケーションコードも存在せず、ユーザーが作成するテキストファイル、エージェントが変換するマークダウンファイルのみがある（格納場所は「コンテンツ構成」に記載）。

## コンテンツ構成

- `メモ/` - ユーザーが作成したテキストファイルを格納する。**エージェントは編集不可**。
- `変換ログ.md` - メモのマークダウン変換履歴を記録するログファイル。
- 以下のカテゴリディレクトリは、メモから変換されたマークダウンファイルの格納先：
  - `情報システム開発/` / `プロジェクトマネジメント/` / `サービスマネジメント/`
  - `システム監査/` / `アルゴリズム/` / `基礎理論/` / `コンピュータシステム/`
  - `データベース/` / `ネットワーク/` / `情報セキュリティ/`
  - `ストラテジ/` / `応用数学/` / `その他/`

## 運用

1. ユーザーが`メモ/`配下に学習用のテキストファイルを格納する。
2. ユーザーの指示によりエージェントがテキストファイルをマークダウンファイルに変換し、適切なディレクトリ配下に格納する。1つのテキストファイルから、セクション単位で複数のマークダウンファイルが生成されうる。
3. ユーザーの依頼を受けて、エージェントが既存のマークダウンファイルに追記または修正する。
4. メモ内に `{...}` で囲まれた記述がある場合は、ユーザーからエージェントへの質問・依頼として扱い、変換時に回答・補足情報を該当箇所に組み込む（詳細は`create-markdown`スキルの「情報追加のポリシー」を参照）。
5. ユーザーの依頼を受けて、エージェントがマークダウンファイルの内容から問題を出題する。

## 使用するスキル

- マークダウンファイルの作成・変更時は `create-markdown` スキルを使用する。
- 出題時は `quiz` スキルを使用する。

## エージェントに対する規定

### 許可される操作

- ファイル・ディレクトリの読み取り（Read、Glob、Grep等のツール）。
- `メモ/`以外のディレクトリへのマークダウンファイルの追加・編集。

### 禁止事項

- `メモ/`配下のファイルの追加・修正・削除（読み取りは許可）。
- gitの操作。
- シェルコマンドの実行。
