# 応用情報技術者試験 学習ノート

IPAの**応用情報技術者試験**（Applied Information Technology Engineer Examination, AP）の学習用プロジェクトです。学習中に取ったテキストメモを、エージェント（Claude Code）が整形してカテゴリ別のマークダウンファイルに変換することで、復習しやすい形に蓄積していきます。

## ディレクトリ構成

```
.
├── メモ/
├── 基礎理論/
├── アルゴリズム/
├── 応用数学/
├── コンピュータシステム/
├── データベース/
├── ネットワーク/
├── 情報セキュリティ/
├── 情報システム開発/
├── プロジェクトマネジメント/
├── サービスマネジメント/
├── システム監査/
├── ストラテジ/
├── その他/
├── 変換ログ.md
├── CLAUDE.md
└── .claude/
    └── skills/
        └── create-markdown/
            └── SKILL.md
```

### カテゴリの内訳

- `メモ/` — 学習中に作成したテキストファイル（変換元）。
- `基礎理論/` — 離散数学、情報理論、計算理論など。
- `アルゴリズム/` — データ構造とアルゴリズム。
- `応用数学/` — 確率、統計、グラフ理論、最適化など。
- `コンピュータシステム/` — プロセッサ、メモリ、システム構成、ソフトウェア。
- `データベース/` — 設計、SQL、トランザクション、正規化。
- `ネットワーク/` — TCP/IP、プロトコル、ネットワーク機器、無線。
- `情報セキュリティ/` — 暗号、認証、攻撃と対策。
- `情報システム開発/` — 要件定義、設計、実装、テスト、開発手法。
- `プロジェクトマネジメント/` — PMBOK、スコープ・コスト・品質・リスク管理。
- `サービスマネジメント/` — ITSMS、ITIL、SLA、運用管理。
- `システム監査/` — 監査計画、実施、報告。
- `ストラテジ/` — 経営戦略、システム戦略、企業活動、法務。
- `その他/` — 上記カテゴリに該当しないもの。

各カテゴリディレクトリには、トピック単位のマークダウンファイルが格納されます（例: `サービスマネジメント/サービスサポート.md`）。

`変換ログ.md` は、`メモ/`配下のテキストファイルがどのマークダウンファイルに変換されたかを記録するログです。エージェントが変換時に参照し、二重変換を防ぐ目的でも使用されます。

## 推奨VSCode拡張機能

マークダウンファイル内のMermaid図（階層構造などのダイアグラム）をVSCodeのプレビューで表示するには、以下の拡張機能をインストールしてください。

```bash
code --install-extension bierner.markdown-mermaid
```

インストール後、`.md`ファイルを開いて `Ctrl+Shift+V` でプレビューを表示すると、GitHub上と同じようにMermaid図が描画されます。

## 運用フロー

1. `メモ/`配下に学習用のテキストファイル（`.txt`）を格納する。
2. エージェントに変換を依頼すると、内容がマークダウンファイルに変換され、適切なカテゴリディレクトリ配下に格納される。
3. 必要に応じて、マークダウンファイルに追記・変更する。

## エージェント連携の前提

このプロジェクトは Claude Code を使用してエージェントと対話することを前提としています。

- マークダウンファイルの作成・編集には`create-markdown`スキルが使用されます。
- スキルの詳細なルール（カテゴリ振り分け・セクション分割・誤り修正・情報追加など）は [`.claude/skills/create-markdown/SKILL.md`](./.claude/skills/create-markdown/SKILL.md) を参照してください。
- エージェントの全体的な振る舞い（許可される操作・禁止事項など）は [`CLAUDE.md`](./CLAUDE.md) で定義されています。

## 依頼例

### テキストメモをマークダウンに変換

```
メモ/サービスマネジメント.txt をマークダウンファイルに変換してください。
```

### 既存のマークダウンファイルに追記

```
サービスマネジメント/サービスサポート.md にインシデント管理の補足を追記してください。
```

### 既存のマークダウンファイルを編集

```
サービスマネジメント/サービスデリバリ.md の SLA の説明を修正してください。
```

### Q&A補完を依頼

メモに `Q. 調査調書とは？` のように質問を残しておくと、変換時に解説（`A. ...`）が補完されます。

### `{}` 記法でエージェントへ依頼

メモ内に `{データフローダイアグラムの例を図で説明してほしい}` のように `{...}` で囲んだ記述を残しておくと、ユーザーからエージェントへの依頼として解釈され、変換時に回答・例・図などが該当箇所に組み込まれます。`{...}` 自体はマークダウンには残りません。
