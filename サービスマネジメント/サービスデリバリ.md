# サービスデリバリ

**シラバス大分類：15. サービスマネジメント**

ITIL の領域の一つで、**中長期的なサービス品質の計画・管理**に関する管理プロセス群。サービスレベル管理、可用性管理、キャパシティ管理、サービス継続管理、財務管理などが含まれる。

## PDCA

継続的改善のためのサイクル。

| ステップ | 内容 |
|---------|------|
| **Plan** | 特定・定義 |
| **Do** | 収集・処理 |
| **Check** | 分析・提示 |
| **Act** | 実装 |

## KGI / KSF / KPI

目標管理に関する指標の階層。

### KGI（Key Goal Indicator／重要目標達成指標）

最終的な具体的ゴール。

例: 今期の売上一億円。

### KSF / CSF（Critical Success Factor／重要成功要因）

KGI 達成に必要な施策や要因。

例: 新規の顧客獲得数を増やす。

### KPI（Key Performance Indicator／重要業績評価指標）

KSF 達成に必要な定量的な目標。

例: リード獲得数 月 1,000 件。

## サービスポートフォリオ

提供を予定・実施・廃止しているサービス全体の管理単位。

```
サービス・パイプライン（検討中） → サービス・カタログ（稼働中） → 廃止したサービス
```

上記すべて（管理するすべてのサービス）を**サービス・ポートフォリオ**と呼ぶ。

サービス・カタログを複数集めたものは**サービスパッケージ**と呼ぶ。

## サービスレベル管理

サービスレベルを定義・合意・測定・改善する管理プロセス。

### SLA（Service Level Agreement／サービスレベル合意）

稼働率の保証など、顧客と合意したサービスレベル。

### SLO（Service Level Objective／サービスレベル目標）

SLA を達成するために設定する内部の目標値。

### SLI（Service Level Indicator／サービスレベル指標）

SLO の達成度を測定する指標。

### SLM（Service Level Management）

SLO を達成するための管理プロセス。

### Q&A

**Q. SLA, SLO, SLI, SLM の具体的な例って何だろう？**

A. EC サイトの可用性管理を例にすると：

- **SLA**: 月間稼働率 99.9% 以上を維持する（顧客との合意）
- **SLO**: 月間ダウンタイム 43 分以内（達成すべき目標値）
- **SLI**: 実測の月間ダウンタイム（分）（測定する指標）
- **SLM**: SLI を継続測定し、SLO 達成に向けて改善活動を行う管理プロセス

**Q. SLA とサービスカタログの違いは？**

A. **サービスカタログ**は「提供しているサービスの一覧」（**何を**提供しているか）。**SLA** は「個別のサービスについて顧客と合意したサービスレベル」（**どのレベルで**提供するか）。カタログがメニュー、SLA がそのメニューの品質保証契約というイメージ。

## サービス可用性管理

SLA で合意された稼働率を提供するための管理。

可用性 = 実際に動いている時間 / 動いているべき時間

※メンテナンスなどの時間は「動いているべき時間」に含めない。

## キャパシティ管理

サーバーやネットワークなどの IT 資産の負荷を管理する。

## サービス継続管理

災害時の影響を少なくし、できるだけサービスを提供できるようにする管理プロセス。

### BCP（Business Continuity Plan／事業継続計画）

災害時にも事業が途切れないようにする計画。

### 復旧目標の指標

| 指標 | 内容 |
|------|------|
| **RLO**（Recovery Level Objective） | どの程度のサービスレベルまで復旧させるか |
| **RPO**（Recovery Point Objective） | どの時点までのデータを復旧させるか |
| **RTO**（Recovery Time Objective） | 災害発生から目標とする復旧完了までに許容される時間 |

## サービス財務管理

サービスを提供するための予算の管理。

### TCO（Total Cost of Ownership）

IT システムの導入から廃止までの総コスト。
