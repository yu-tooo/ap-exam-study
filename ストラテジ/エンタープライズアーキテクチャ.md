# エンタープライズアーキテクチャ

**シラバス大分類：17. システム戦略**

**エンタープライズアーキテクチャ**（Enterprise Architecture, EA）は、組織全体の業務とシステムを統一的に整理し、**全体最適**の観点から業務を改善する仕組み。現状から理想までを段階的に設計するためのモデル群を持つ。

## 3つのモデル

| モデル | 役割 |
|---|---|
| **現状モデル**（As-Is） | 現状の業務・システムを整理 |
| **次期モデル** | 現状から理想に至る途中段階。短期〜中期で実現可能な姿 |
| **理想モデル**（To-Be） | あるべき将来像 |

各モデルは下記**4つの体系**で表現する。

- **政策・業務体系**
- **データ体系**
- **適用処理体系**（アプリケーション）
- **技術体系**（テクノロジー）

## 4つのアーキテクチャ（次期モデルの構成要素）

上位から順に着手するのが原則（上位の方針を決めないと下位の設計がぶれるため）。

### 1. ビジネスアーキテクチャ（BA）

業務のあり方を整理する層。

- **参照モデル**: 政策・業務参照モデル（BRM）、業績測定参照モデル（PRM）
- **成果物例**: 業務フローの整理、役割分担表

### 2. データアーキテクチャ（DA）

業務で扱うデータの構造と流れを整理する層。

- **参照モデル**: データ参照モデル（DRM）
- **成果物例**: データフロー図、共通データベースの構築、アクセス権限の設計

### 3. アプリケーションアーキテクチャ（AA）

業務を支えるアプリケーションの構成を整理する層。

- **成果物例**: パッケージ図、アクティビティ図
- **アプリ例**: 模擬店管理システム、ステージ進行アプリ、広報用SNSツール

### 4. テクノロジーアーキテクチャ（TA）

アプリを動かす基盤技術を整理する層。

- **成果物例**: ネットワーク構成図
- **施策例**: Wi-Fi設備の整備、バックアップ体制の構築

## 全体の流れ

```
As-Is（現状モデル）
   ↓ 課題・ギャップを把握
次期モデル（BA → DA → AA → TA の順で設計）
   ↓ 段階的に移行
To-Be（理想モデル）
```
