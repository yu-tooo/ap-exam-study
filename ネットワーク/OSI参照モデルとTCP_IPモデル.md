# OSI参照モデルとTCP/IPモデル

**シラバス大分類：10. ネットワーク**

## OSI参照モデル

ネットワーク通信を**7階層**に分けて整理した国際標準のモデル。各層が独立した役割を持ち、上位層は下位層のサービスを利用する。

| 層 | 名称 | 役割 |
|----|------|------|
| 第7層 | **アプリケーション層** | ユーザーに直接関わる通信サービス（Webブラウザ・メールなど）を提供する。 |
| 第6層 | **プレゼンテーション層** | データの形式変換、暗号化や圧縮を行う。 |
| 第5層 | **セッション層** | アプリケーション間でセッションの確立・維持・終了を管理する。 |
| 第4層 | **トランスポート層** | データの信頼性確保、順序管理、再送制御などを行う。 |
| 第3層 | **ネットワーク層** | 異なるネットワーク間でのパケットの経路制御（ルーティング）を行う。 |
| 第2層 | **データリンク層** | 隣接するノード間のデータ転送と誤り検出を行う。 |
| 第1層 | **物理層** | 通信ケーブルなど、物理的な信号の伝送を担う。 |

## カプセル化

データは**送信側で上位層→下位層**へと順番に降りながら、各層で必要なヘッダを付け加えていく。受信側ではその逆順にヘッダを取り除いていく。

```
[送信側]                              [受信側]
アプリ層    : [データ]                  : [データ]
プレゼン層  : [P|データ]                : [P|データ]
セッション層: [S|P|データ]              : [S|P|データ]
トランスポ層: [T|S|P|データ]            : [T|S|P|データ]
ネットワク層: [N|T|S|P|データ]          : [N|T|S|P|データ]
データリンク: [D|N|T|S|P|データ|F]      : [D|N|T|S|P|データ|F]
物理層      : 0010110... （ビット列）
                  ↓ ケーブル等で伝送 ↑
```

## TCP/IPモデル

実際のインターネット通信で広く使われている**4階層**のモデル。OSI参照モデルの複数層をまとめた構成になっている。

| TCP/IPモデル | 対応するOSI参照モデル | 主な役割・プロトコル |
|------------|--------------------|------------------|
| **アプリケーション層** | 第5〜7層 | HTTP・FTP・SMTPなど、各種サービスの通信。 |
| **トランスポート層** | 第4層 | TCP・UDPによるデータ転送制御。 |
| **インターネット層** | 第3層 | IPによるパケット送信・ルーティング。L3スイッチによるLAN内部の高速データ通信。 |
| **ネットワークインターフェース層** | 第1〜2層 | MACアドレスを使った隣接ノード間の通信。スイッチングハブなど。 |

### 代表的な機器（TCP/IP上位層付近）

- **ゲートウェイ**: 異なるプロトコル間の中継・変換を行う機器。
- **プロキシサーバ**: クライアントとサーバの間で通信を中継する。キャッシュ・アクセス制御・匿名化などの目的で使われる。
- **ファイアウォール**: 通信を監視・制御し、不正アクセスから内部ネットワークを守る。

## ヘッダ構造（イーサネットフレーム）

データリンク層で送受信されるイーサネットフレームの構造。

```
[イーサネットヘッダ] [IPヘッダ] [TCP/UDPヘッダ] [データ] [FCS]
```

- **イーサネットヘッダ**: 隣接ノード間の通信に使うMACアドレス情報など。
- **IPヘッダ**: 送信元・宛先のIPアドレスなど。
- **TCP/UDPヘッダ**: ポート番号やシーケンス番号など。
- **データ**: アプリケーション層からのペイロード。
- **FCS**（Frame Check Sequence）: フレーム末尾に付けられる誤り検出用のチェックビット列。
