# アプリケーション層

**シラバス大分類：10. ネットワーク**

ユーザーが直接利用するアプリケーション間の通信を担う層。各種プロトコルを通じて、Web・メール・ファイル転送などのサービスにアクセスできる。

## 主なプロトコルとポート番号

| プロトコル | ポート番号 | 用途 |
|-----------|----------|------|
| **HTTP** | 80 | Webページの転送（平文） |
| **HTTPS** | 443 | HTTPの暗号化版（TLS／SSLで保護） |
| **DHCP** | 67（サーバ）／68（クライアント） | IPアドレス・サブネットマスクなどを自動付与 |
| **DNS** | 53 | ドメイン名 ⇔ IPアドレスの変換 |
| **FTP** | 20（データ）／21（制御） | サーバへのファイル転送 |
| **SMTP** | 25／587 | メールの送信・転送 |
| **POP3** | 110 | メールサーバからメールを取得（取得後はサーバから削除） |
| **IMAP** | 143 | メールをサーバ上で管理しながら閲覧（複数端末で同期可能） |

## DHCP（Dynamic Host Configuration Protocol）

新しくネットワークに参加したコンピュータに、IPアドレス等のネットワーク設定を**自動で割り当てる**プロトコル。

### 動作の流れ

1. **DISCOVER**: 新しくネットワークに入ったコンピュータが**ブロードキャスト**でDHCPサーバを探す。
2. **OFFER**: DHCPサーバが応答し、IPアドレス候補を**提案**する。
3. **REQUEST**: クライアントが提案されたIPアドレスを使いたい旨を**ブロードキャスト**でリクエストする。
4. **ACK**: DHCPサーバが正式に**割り当て**を確定する。

```
クライアント              DHCPサーバ
    │                         │
    │── ① DISCOVER（ブロードキャスト） ─▶│
    │                         │
    │◀── ② OFFER（IP候補を提案） ──────│
    │                         │
    │── ③ REQUEST（このIPで！） ──────▶│
    │                         │
    │◀── ④ ACK（OK、割当確定） ────────│
```

## DNS（Domain Name System）

ドメイン名（例: `example.com`）を**IPアドレスに変換**する仕組み。階層的な分散データベースとして実装されており、世界中のDNSサーバが連携して名前解決を行う。

## FTP（File Transfer Protocol）

サーバとクライアント間でファイルを転送するプロトコル。**2つのコネクション**を使うのが特徴。

| コネクション | ポート番号 | 役割 |
|------------|----------|------|
| **コントロールコネクション** | **21** | コマンドや応答（接続制御）をやり取り |
| **データコネクション** | **20** | 実際のファイルデータを転送 |

> 元メモではポート番号の対応が逆になっていたため修正している（コントロール=21、データ=20が正しい）。

## メールプロトコルの整理

| 役割 | プロトコル |
|------|-----------|
| メール**送信**（クライアント→サーバ、サーバ間転送） | **SMTP** |
| メール**取得**（サーバ→クライアント・取得後にサーバから削除） | **POP3** |
| メール**閲覧**（サーバ上で管理、複数端末で同期） | **IMAP** |

```
[送信]              [受信]
クライアント         クライアント
    │                  ▲
   SMTP               POP3 / IMAP
    ▼                  │
[送信メールサーバ] ─SMTP→ [受信メールサーバ]
```
