# データリンク層

隣接するノード間のデータ転送と誤り検出を行う層。**MACアドレス**を用いて同一ネットワーク内の機器を識別する。

## ノード

ネットワークに接続される個々の機器（パソコン、スマホ、ルータなど）の総称。

## MACアドレス

ネットワーク機器を一意に識別するための**48ビット**（6バイト）のアドレス。NICに製造時に割り当てられる。

```
| OUI（24ビット） | 製造番号（24ビット） |
   ↑                ↑
   ベンダーID         ベンダー内での連番
```

- **上位24ビット**: ベンダーごとに割り当てられる識別ID（OUI）。
- **下位24ビット**: ベンダー内での製造番号。

## イーサネットヘッダ

データリンク層で付与されるヘッダ。**全14バイト**。

| フィールド | サイズ | 内容 |
|-----------|-------|------|
| 宛先MACアドレス | 6バイト | 送信先機器のMACアドレス |
| 送信元MACアドレス | 6バイト | 送信元機器のMACアドレス |
| タイプ | 2バイト | 上位層のプロトコル種別（例: IPv4 = 0x0800、ARP = 0x0806） |

## データリンク層の機器

| 機器 | 役割 |
|------|------|
| **スイッチングハブ** | 宛先MACアドレスを見て、**目的のLANポートにだけ**データを流す。学習機能でポート⇔MACアドレス対応表（MACアドレステーブル）を構築する。 |
| **ブリッジ** | スイッチングハブと同様にMACアドレスでフィルタリングするが、ソフトウェア処理が中心で性能が低い。現在はほぼスイッチングハブに置き換わっている。 |

## リンクアグリゲーション

複数の物理回線を**1つの論理的な回線**として扱う技術。「アグリゲーション＝束ねる」の意味。

- 帯域幅を増やせる（例: 1Gbps × 4本 → 4Gbps相当）
- 1本が切れても残りの回線で通信を継続できる（冗長化）

## VLAN（Virtual LAN）

物理的なネットワーク構成にとらわれず、**論理的に**ネットワークをグルーピングする技術。

- ブロードキャストの範囲を制限し、ネットワーク負荷やセキュリティリスクを軽減できる。
- 同じVLAN内は同一ネットワークとして扱える（ルータを介さず通信可能）。

### ポートベースVLAN

スイッチングハブの**ポートごとにVLAN ID**を割り当て、同じVLAN IDのポート同士でグループを作る方式。最も基本的なVLAN構成。

## スパニングツリー

ネットワーク内の**ループ（フレームの永久周回）を防ぐ**プロトコル。

- 物理的にループ構造でも、一部のポートを**ブロッキングポート**として仮想的に切断し、論理的な**ツリー構造**にする。
- いずれかの経路に障害が発生したときは、ブロッキングポートを有効化して経路を切り替える（自動冗長）。

```
[物理構成]              [スパニングツリー適用後]
SW1 ──── SW2            SW1 ──── SW2
 │        │              │
 └─ SW3 ──┘              └─ SW3 ─×─ （×：ブロッキング）
   ループ発生              ツリー化
```
