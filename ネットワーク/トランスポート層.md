# トランスポート層

データを確実に・効率的に届けるための制御を行う層。アプリケーションごとに通信を識別する**ポート番号**もこの層で扱う。

## TCP と UDP の比較

| 観点 | **TCP**（Transmission Control Protocol） | **UDP**（User Datagram Protocol） |
|------|----------------------------------------|--------------------------------|
| 通信方式 | コネクション型（接続を確立してから通信） | コネクションレス型（接続なしで送りっぱなし） |
| 信頼性 | 高い（再送・順序保証・誤り検出あり） | 低い（再送・順序保証なし） |
| 速度 | 遅い | 速い |
| ヘッダオーバヘッド | 大きい | 小さい |
| 用途 | Web（HTTP）、メール、ファイル転送など信頼性重視 | 動画・音声配信、DNS、オンラインゲームなど速度重視 |

## ポート番号

トランスポート層が**アプリケーションを識別する**ための番号。1つのコンピュータ上で複数の通信を同時に行えるようにする。16ビット（0〜65535）で表す。

| 範囲 | 用途 |
|------|------|
| 0〜1023 | **ウェルノウンポート**（HTTP=80など、主要プロトコルに予約） |
| 1024〜49151 | 登録済みポート |
| 49152〜65535 | 動的・プライベートポート（クライアント側で一時的に使用） |

## 3wayハンドシェイク

TCPがコネクションを確立する際の**3段階の手順**。これによりクライアントとサーバ双方が「相手が応答可能であること」を確認する。

### 流れ

1. **クライアント → サーバ（SYN）**: クライアントが初期シーケンス番号を送る。
2. **サーバ → クライアント（SYN+ACK）**: サーバが受け取ったシーケンス番号に**1**を足した値を**確認応答番号**として送り、同時に**サーバ側のシーケンス番号**も送る。
3. **クライアント → サーバ（ACK）**: クライアントがサーバから受け取ったシーケンス番号に**1**を足した確認応答番号を送る。

```
クライアント                    サーバ
    │                             │
    │ ─── SYN (seq=x) ──────────▶│   ①
    │                             │
    │◀──── SYN+ACK ──────────────│   ②
    │      (seq=y, ack=x+1)      │
    │                             │
    │ ─── ACK (ack=y+1) ────────▶│   ③
    │                             │
    │     コネクション確立完了     │
    │═══════════════════════════│
```

これにより双方向の通信が可能であることが確認できる。

UDPには**シーケンス番号がない**ため、3wayハンドシェイクも行われない（コネクションレス型のため）。
