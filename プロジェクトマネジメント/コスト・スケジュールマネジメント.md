# コスト・スケジュールマネジメント

**シラバス大分類：14. プロジェクトマネジメント**

## コストマネジメント

予算の作成と管理を行う活動。見積もりによる予算策定と、EVM分析による進捗・コスト評価が中心。

### 見積もり手法

| 手法 | 概要 |
|------|------|
| **類推法** | 過去の類似プロジェクトを参考に見積もる。簡便だが精度は粗い。 |
| **プログラムステップ法** | 記述するプログラムの行数（LOC）を基に見積もる。 |
| **パラメトリック見積**（係数モデル法） | 過去データを基に、特定のパラメータと数式を用いて精度の高い見積もりを行う。 |
| **積算法**（ボトムアップ見積） | 作業項目（アクティビティ）ごとに見積もり、それらを合計する。詳細だが工数がかかる。 |
| **3点見積もり法** | **楽観値**・**悲観値**・**最頻値**の3つの値を基に見積もる。期待値 = (楽観 + 4×最頻 + 悲観) / 6（PERT）。 |
| **COCOMO** | 予想されるプログラムの行数に、開発メンバーの能力や求められる信頼性などの係数を掛けて見積もる。システム開発以外の分野でも用いられる。 |
| **ファンクションポイント法** | 利用者から見える画面や帳票などの**機能の難易度や個数**に着目して見積もる。 |

### EVM分析（Earned Value Management）

予算とスケジュールの両方の観点からプロジェクトの進捗を評価する手法。

#### 基本値（測定値）

| 略語 | 名称 | 意味 |
|------|------|------|
| **PV**（Planned Value） | 計画価値 | ある時点で**計画上**完了しているはずの作業の予算。計画作業に割り当てられた予算額。 |
| **EV**（Earned Value） | 出来高 | ある時点で**実際に完了した作業**の出来高を、予算ベースで評価した値。 |
| **AC**（Actual Cost） | 実コスト | 完了した作業に対して**実際に発生したコスト**。 |
| **BAC**（Budget At Completion） | 完成時総予算 | プロジェクト全体に割り当てられた総予算。 |

#### 差異・効率指数

| 略語 | 名称 | 計算式 | 評価 |
|------|------|--------|------|
| **CV**（Cost Variance） | コスト差異 | `CV = EV − AC` | 正＝予算内、負＝予算超過 |
| **SV**（Schedule Variance） | スケジュール差異 | `SV = EV − PV` | 正＝進捗が計画より進んでいる、負＝遅延 |
| **CPI**（Cost Performance Index） | コスト効率指数 | `CPI = EV / AC` | 1以上＝予算内、1未満＝予算超過 |
| **SPI**（Schedule Performance Index） | スケジュール効率指数 | `SPI = EV / PV` | 1以上＝計画通り以上、1未満＝遅延 |

#### EVMのグラフによる説明

横軸を時間、縦軸を金額（コスト）として、PV・EV・ACの3本の曲線を描く。現時点（評価時点）における3本の値の差から、コストとスケジュールの状態を読み取る。

```
コスト
  ▲
  │                                   ●BAC（完成時総予算）
  │                              ／
  │                         ／
  │                  AC ／    ← 実コスト
  │              ／  ╱
  │           ／  ╱  PV       ← 計画価値
  │        ／  ╱   ╱
  │     ／   ╱   ╱           ↑ CV = EV − AC（負＝予算超過）
  │   ／  ╱   ╱   EV         ↑ SV = EV − PV（負＝遅延）
  │ ╱   ╱   ╱
  └──────────────────────────▶ 時間
              ↑現時点
```

##### 読み取り例

ある時点で `PV = 100`, `EV = 80`, `AC = 120` の場合：

- `CV = EV − AC = 80 − 120 = −40` → **40万円の予算超過**
- `SV = EV − PV = 80 − 100 = −20` → **20万円分のスケジュール遅延**
- `CPI = 80 / 120 ≒ 0.67` → 1未満なので**コスト効率が悪い**
- `SPI = 80 / 100 = 0.8` → 1未満なので**スケジュールが遅れている**

このようにEVMはコストとスケジュールを統合的に把握できるため、プロジェクトの早期警告に有効。

## スケジュールマネジメント

スケジュールを作成し管理する活動。アクティビティの依存関係を整理し、所要期間と完了予定を可視化する。

### 主な技法

- **ガントチャート**: 横軸を時間、縦軸をタスクとして棒で表現。進捗の可視化に優れる。依存関係の表現は弱い。
- **アローダイアグラム**（PERT図）: 矢印でアクティビティ、丸印でイベント（マイルストーン）を表す。クリティカルパスの算出に用いる。
- **プレシデンスダイアグラム**（PDM, Precedence Diagram Method）: ノードでアクティビティを、矢印で依存関係を表す。4種類の依存関係を表現できる。

### プレシデンスダイアグラム

各アクティビティをノード（ボックス）で表し、ノード間の矢印で**依存関係**（先行関係）を示す。後続作業をいつ開始／終了できるかを4種類の関係で表現する。

#### 4つの依存関係

| 略号 | 名称 | 意味 |
|------|------|------|
| **FS**（Finish-to-Start） | 終了→開始 | 先行が**終了**したら後続を**開始**できる。最も一般的な関係。 |
| **FF**（Finish-to-Finish） | 終了→終了 | 先行が**終了**したら後続を**終了**できる。 |
| **SS**（Start-to-Start） | 開始→開始 | 先行が**開始**したら後続を**開始**できる。並行作業に使う。 |
| **SF**（Start-to-Finish） | 開始→終了 | 先行が**開始**したら後続を**終了**できる。最も稀な関係。 |

#### 図解

```
FS（終了→開始）        FF（終了→終了）
┌────┐ FS ┌────┐      ┌────┐ FF ┌────┐
│ A  │───▶│ B  │      │ A  │──▶ │ B  │
└────┘    └────┘      └────┘    └────┘
Aが終わったらB開始     Aが終わったらB終了

SS（開始→開始）        SF（開始→終了）
┌────┐ SS ┌────┐      ┌────┐ SF ┌────┐
│ A  │───▶│ B  │      │ A  │───▶│ B  │
└────┘    └────┘      └────┘    └────┘
Aが始まったらB開始     Aが始まったらB終了
```

##### 各関係の例

- **FS**: 「設計が終わったら製造を開始する」
- **FF**: 「テスト実施が終わったらテスト報告書作成も終わらせる」（テストと報告書を並行で進めつつ、最終的に同じタイミングで完了させる）
- **SS**: 「データ移行を開始したら、移行ログの監視も開始する」
- **SF**: 「新システムの稼働を開始したら、旧システムの稼働を終了する」（システム切替）

### スケジュール短縮手法

予定より遅れている、もしくは納期を前倒しにする必要があるときに用いる。

- **クラッシング**: リソース（人員・設備）を追加投入することで作業期間を短縮する。コスト増のトレードオフがある。
- **ファストトラッキング**: 本来は順次実施する作業を、可能な範囲で**並列**で実施する。手戻りリスクが増える。
