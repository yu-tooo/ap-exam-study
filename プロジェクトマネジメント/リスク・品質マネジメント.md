# リスク・品質マネジメント

## リスクマネジメント

リスク（負の不確実性）と**チャンス**（正の不確実性）の両方をコントロールする活動。

### チャンス（好機）への対応戦略

| 戦略 | 内容 |
|------|------|
| **活用** | チャンスの発生確率を100%にすることを目指す。 |
| **共有** | 第三者と組んでチャンスを獲得する。 |
| **強化** | チャンスの発生確率を上げる、またはチャンスの影響を増やす。 |
| **受容** | 何もしない（特別な行動を起こさず受け入れる）。 |

> 元メモでは「活動」と記載されていたが、PMBOKの正式用語は「**活用**（Exploit）」のため修正している。

### リスク（悪影響）への対応戦略

| 戦略 | 内容 |
|------|------|
| **回避** | リスクの発生確率を0%にすることを目指す（計画変更などでリスク要因を取り除く）。 |
| **転嫁** | 悪影響を第三者に移し、リスクの影響を下げる（保険・契約による責任移転など）。 |
| **軽減** | リスクの発生確率を下げる、またはリスクの影響を減らす。 |
| **受容** | 何もしない（コンティンジェンシー予備で備えるのみ）。 |

チャンス／リスクの戦略は「**受容**」が共通で、それ以外は鏡像の関係（活用↔回避、共有↔転嫁、強化↔軽減）になっている。

### RBS（Risk Breakdown Structure）

プロジェクトに潜在するリスクを体系的に分類・整理した階層図。WBSのリスク版。

> **注意**: 同じ「RBS」と略されるものに **Resource Breakdown Structure**（リソース・ブレークダウン・ストラクチャー：資源マネジメントで使う、人的・物的資源の階層分解）があるので混同に注意。

## 品質マネジメント

プロジェクトや製品の品質を向上させる活動。品質計画・品質保証・品質管理の3プロセスからなる。代表的なツールに**QC7つ道具**と**新QC7つ道具**がある。

### QC7つ道具

**定量データ**（数値）を分析するためのツール群。製造現場の品質管理から発展したもので、すでに発生した事象を統計的に分析するのに向く。

- **パレート図**（棒グラフ＆折れ線グラフ）: 項目を頻度順に並べた棒グラフと、累積比率の折れ線を重ねた図。重点管理項目の特定に使う（パレートの法則）。
- **散布図**: 2つの変数の相関関係を見る。
- **特性要因図**（フィッシュボーン／石川ダイアグラム）: 結果（特性）に影響する要因を魚の骨状に整理した図。原因分析に使う。
- **管理図**: 工程の時間的変化を上限・下限の管理線とともに表示し、異常を検出する。
- **ヒストグラム**: データのばらつきを階級別の度数分布で表す。
- **チェックシート**: データの収集・点検を効率化する記録用紙。
- **層別**: データを属性（機械・作業者・時間帯など）でグループ分けして傾向を見る分析手法。

#### 新QC7つ道具と比較したときの特徴

- **定量データ**（数値）を扱う。
- **既に発生した事象の分析**（事後分析）に強い。
- 統計的・数値的アプローチで、客観的な現状把握に適する。
- 主に**製造工程**の管理で発展した。

### 新QC7つ道具

**定性データ**（言語データ）を整理・構造化するためのツール群。数値化しにくい問題や、これから取り組む計画立案に向く。

- **親和図法**: 多様な言語データをグループ化して、問題の構造を整理する（KJ法に類似）。
- **連関図法**: 原因と結果の関係を矢印でつなぎ、複雑に絡み合った要因関係を可視化する。
- **系統図法**: 目的を達成するための手段を階層的に展開する（目的→手段の樹形図）。
- **PDPC法**（Process Decision Program Chart）: 計画実行時に予想される事態と対応策を樹形図で表す。リスクを織り込んだ計画立案に使う。
- **アローダイアグラム**: 作業の順序関係を矢印で表現する。スケジュール管理にも用いられる（PERT図）。
- **マトリックス図法**: 2つ以上の要素の関係を行列形式で整理する。
- **マトリックスデータ解析法**: マトリックス図のデータを多変量解析で数値的に処理する（新QC7つ道具の中で唯一の定量手法）。

#### QC7つ道具と比較したときの特徴

- **定性データ**（言語データ）を扱う（マトリックスデータ解析法のみ例外）。
- **これから発生する事象の予測・計画**（事前検討）に強い。
- 構造化・図解化のアプローチで、関係者の合意形成にも使える。
- 主に**企画・設計段階**での問題整理に用いられる。

### QC7つ道具と新QC7つ道具の対比

| 観点 | QC7つ道具 | 新QC7つ道具 |
|------|-----------|-------------|
| 扱うデータ | 定量（数値） | 定性（言語） |
| 適用フェーズ | 事後分析（現状把握・原因分析） | 事前検討（計画立案・問題整理） |
| 主な利用場面 | 製造工程の品質管理 | 企画・設計段階の合意形成 |
| アプローチ | 統計的・数値的 | 構造化・図解的 |
