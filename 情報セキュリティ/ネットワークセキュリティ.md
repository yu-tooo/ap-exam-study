# ネットワークセキュリティ

## ファイアウォール

パケットの中身（送信元アドレス・宛先アドレス・ポート番号など）を見て、**通すか通さないかを判断**する機器。組織内ネットワークの境界に配置し、不正なアクセスを遮断する。

## DMZ（DeMilitarized Zone、非武装地帯）

**外部から直接アクセスできるネットワーク領域**。インターネットと内部ネットワークの中間に配置する。

- DMZには **Webサーバ**や **DNSサーバ**など、**公開する必要があるサーバだけ**を配置する。
- **DBサーバ**など機密情報を扱うサーバは DMZ には置かず、内部ネットワーク側に配置する。
- DMZの境界には**ファイアウォール**を設置する（通常は2段階のファイアウォール構成）。

```
[インターネット]
       │
   [ファイアウォール1]
       │
   ┌──[DMZ]──┐
   │ Webサーバ │ ← 外部公開OK
   │ DNSサーバ │
   └─────────┘
       │
   [ファイアウォール2]
       │
   [内部ネットワーク]
       │
    DBサーバ等  ← 外部からは隔離
```

## プロキシサーバ

**通信を中継するサーバ**。クライアントの代わりに外部と通信する。

- **社内PCを外部から隠蔽**できる（IPアドレスを秘匿）。
- プロキシサーバ**にだけ**高度なセキュリティを集中して確保すればよい。
- **キャッシュ**を持たせることで、同じコンテンツへの繰り返しアクセスを高速化できる。

## IDPS（Intrusion Detection and Prevention System）

不正なアクセスや攻撃を**検出・防止**するためのシステム。検出のみか防止まで行うかで分かれる。

| 種類 | 略 | 役割 |
|------|----|------|
| **IDS**（Intrusion Detection System） | 侵入検知 | 怪しい通信を**検出して通知**する。遮断は行わない。 |
| **IPS**（Intrusion Prevention System） | 侵入防止 | 怪しい通信を**自動的に遮断**する。 |

## SIEM（Security Information and Event Management）

ファイアウォール・プロキシサーバ・ルータなどの**ログを集約・相関分析**し、脅威を検出・対応するシステム。複数機器のログを横串で監視できる。

## WAF（Web Application Firewall）

**Webアプリケーション**への攻撃を検出・防止するシステム。HTTPリクエストの**データ部まで解析**するため、SQLインジェクションやクロスサイトスクリプティングなどアプリ層の攻撃に有効。

## DLP（Data Loss Prevention）

**機密情報の流出や消失を防ぐ**ためのセキュリティシステム。機密情報を識別し、その持ち出し・送信などを制御する。「**機密情報だけに厳しい制御を適用する**」のが特徴。

## UTM（Unified Threat Management、統合脅威管理）

ファイアウォール・IPS・WAF・アンチウイルスなど、**複数のセキュリティ機能を1つの装置で提供**する。中小企業で導入しやすい。

## コンテンツフィルタリング

**有害なWebサイトや不適切なコンテンツへのアクセスをブロック**する仕組み。教育機関や企業の端末で利用される。

## MACアドレスフィルタリング

ネットワークへの**接続を許可する端末のMACアドレス**だけを登録しておき、未登録の端末は接続させない仕組み。MACアドレスは偽装可能なため、これ単独ではセキュリティ強度は限定的。
