# マルウェア

> **Q. マルウェアって？**
>
> A. **マルウェア**（Malware）は **Mal**icious Soft**ware** の略。利用者や組織に**害を与える目的**で作られたソフトウェアの総称。ウイルス・ワーム・スパイウェア・トロイの木馬・ランサムウェアなど、用途や挙動によって分類される。

## マルウェアの種類

| 種類 | 概要 |
|------|------|
| **ウイルス** | システムのファイルを削除・改ざんしたり、意図しない画面表示を行ったりする。他のプログラムに**寄生**して感染を広げる。 |
| **ワーム** | **ネットワーク経由で他のコンピュータに自分自身をコピー**して感染を広げる。寄生先を必要としない自己完結型。 |
| **スパイウェア** | システムに入り込んで**個人情報を抜き取り**、外部に送信する。 |
| **トロイの木馬** | 正常なソフトウェアに見せかけて、裏で不正な処理を行う。自己増殖はしない。 |
| **ランサムウェア** | **データを暗号化して使用不能にし**、復号と引き換えに金品（身代金）を要求する。 |

## 検出回避手法

マルウェアがウイルス対策ソフトの検出から逃れるための手法。

| 手法 | 概要 |
|------|------|
| **ポリモーフィック型** | プログラム本体を**暗号化**することで、シグネチャ（パターン）によるパターンマッチングを回避する。 |
| **メタモーフィック型** | 動作に変化がない範囲でプログラムの**一部を書き換え**、シグネチャによるパターンマッチングを回避する。 |

## マルウェアの検出方式（EDR等で利用）

**EDR**（Endpoint Detection and Response）は、端末上で脅威を検出し、対応するためのセキュリティ機能。以下の検出方式を組み合わせて使う。

| 方式 | 検出ロジック |
|------|------------|
| **コンペア法** | データやプログラム**そのものを比較**して不正を検出する。 |
| **チェックサム法** | ある単純な計算で求めた**チェックサム値を比較**する。 |
| **パターンマッチング法** | **シグネチャファイル**（ウイルス定義ファイル）と照合してマルウェアを検出する。 |
| **ヒューリスティック法** | マルウェアの**挙動パターン**をチェックして検出する（既知のシグネチャに依らない）。 |
| **動的ヒューリスティック法** | 感染しても問題ない**サンドボックス環境**で実際に動かして挙動を検証する。 |
