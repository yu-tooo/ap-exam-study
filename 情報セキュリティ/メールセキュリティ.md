# メールセキュリティ

**シラバス大分類：11. セキュリティ**

メールの送受信に関するセキュリティ技術。なりすまし・盗聴・改ざんから守る。

## OP25B（Outbound Port 25 Blocking）

**ISPが、自社の動的IPアドレス帯から外部の25番ポート（SMTP）への通信を遮断する**仕組み。ボット感染端末などからのスパム送信を防ぐ目的で導入された。

- 通常のSMTP（25番）を使うと、自宅から外部メールサーバへ直接送信できなくなる。
- 代わりに**サブミッションポート（587番）**を使うことで、SMTP-AUTHによるユーザー認証を経て送信できる。

## サブミッションポート（587番）

**SMTP-AUTHでユーザー認証**することで、メール送信を許可するポート。OP25Bの導入後、メール送信クライアントは587番＋SMTP-AUTHを使うのが一般的。

## S/MIME（Secure / Multipurpose Internet Mail Extensions）

メールの**盗聴・なりすまし・改ざん**を防止するための技術。

- **認証局（CA）**が発行する電子証明書を使う。
- メールの**暗号化**には公開鍵暗号方式を、**改ざん検知・送信者認証**にはデジタル署名を使う。

## 送信ドメイン認証技術（SPF・DKIM・DMARC）

送信元メールサーバの正当性を検証することで、なりすましメール（フィッシング・スパム）を防ぐ仕組み。

### SPF（Sender Policy Framework）

**送信元メールサーバの正当性を確認**し、不正メールを防ぐ。

- 送信ドメインの管理者が、**正規の送信元IPアドレスをDNSに事前登録**する。
- 受信側は、メールの送信者IPアドレスとDNSに登録されたIPアドレスを照合する。

### DKIM（DomainKeys Identified Mail）

**デジタル署名**でメールの正当性を検証し、不正メールを防ぐ。

- 送信側がメールに秘密鍵で署名を付与する。
- 受信側がDNSから公開鍵を取得して署名を検証する。

### DMARC（Domain-based Message Authentication, Reporting and Conformance）

**SPFやDKIMの検証結果に基づき、不正メールの処理方針を指定**する仕組み。

- ドメイン管理者が **DNS** に処理方針を公開する。
- 受信側は、SPF／DKIMの検証に失敗したメールに対して、方針通りの処理を行う。

#### DMARC の処理方針

| 方針 | 内容 |
|------|------|
| **none** | 何もしない（モニタリングのみ） |
| **quarantine** | **隔離**する（迷惑メールフォルダに振り分けるなど） |
| **reject** | **ブロック**（受信拒否）する |

## まとめ

| 技術 | 主な目的 | 仕組み |
|------|--------|------|
| **OP25B** | スパム送信防止 | 25番ポートの送信遮断 |
| **サブミッションポート** | 正規送信の許可 | 587番＋SMTP-AUTH |
| **S/MIME** | 盗聴・なりすまし・改ざん防止 | 認証局＋公開鍵暗号＋デジタル署名 |
| **SPF** | なりすまし防止 | 送信元IPアドレスの照合 |
| **DKIM** | なりすまし・改ざん防止 | デジタル署名 |
| **DMARC** | SPF/DKIM結果の処理方針 | DNSで方針を公開 |
