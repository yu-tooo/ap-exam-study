# 暗号化と電子証明書

**シラバス大分類：11. セキュリティ**

## 暗号化方式

### 共通鍵暗号方式

暗号化と復号で**同じ鍵**を使う方式。処理が速いが、鍵を安全に共有する必要がある。

| アルゴリズム | 特徴 |
|-------------|------|
| **AES**（Advanced Encryption Standard） | 現在の標準的な共通鍵暗号アルゴリズム。鍵長128/192/256ビット。 |

### 公開鍵暗号方式

暗号化と復号で**異なる鍵（公開鍵と秘密鍵）**を使う方式。鍵共有が容易だが、処理は遅い。

| アルゴリズム | 安全性の根拠 |
|-------------|------------|
| **RSA** | **素因数分解の困難さ**を安全性の根拠とする。 |
| **ECC**（Elliptic Curve Cryptography、楕円曲線暗号） | **楕円曲線上の離散対数問題の困難さ**を安全性の根拠とする。RSAより短い鍵長で同等の安全性が得られる。 |

### ハイブリッド暗号

**共通鍵暗号と公開鍵暗号を組み合わせた**方式。SSL/TLS で使われる。

1. データの暗号化には**高速な共通鍵暗号**を使う。
2. その共通鍵を相手に渡すときに**公開鍵暗号で暗号化**する（鍵共有問題を解決）。

これにより「速度」と「安全な鍵共有」の両立が可能になる。

## 暗号化応用技術

| 技術 | 概要 |
|------|------|
| **SSL/TLS** | インターネット上でデータを暗号化して送受信するプロトコル。**ハイブリッド暗号**が使われる。HTTPS の基盤。 |
| **IPsec** | **パケットを暗号化**するプロトコル。IPv4 ではオプションだが、IPv6 では標準的に組み込まれている。VPN の構築によく使われる。 |
| **SSH**（Secure Shell） | リモートサーバとの通信を暗号化するプロトコル。リモートログインやファイル転送（SFTP）に使われる。 |

## デジタル証明書（電子証明書）

**公開鍵の正当性を証明する**ファイル。「この公開鍵は確かにこの組織のものである」と認証局が保証する。

### 発行の流れ

1. 申請者が **CSR**（Certificate Signing Request）を作成する。
   - CSRには**組織名・所在地・公開鍵**などが含まれる。
2. CSRを**認証局（CA）**に送付する。
3. 認証局が審査・登録し、**デジタル証明書を作成**する。認証局の秘密鍵で署名（暗号化）してサーバに渡す。
4. クライアントがサーバに接続するときに、サーバから証明書を受け取り、**認証局の公開鍵で検証**する。

```
[サーバ]                    [認証局 CA]              [クライアント]
   │                            │                        │
   │── ① CSR（公開鍵+組織情報）──▶│                        │
   │                            │                        │
   │◀── ② 証明書（CA秘密鍵で署名）─│                        │
   │                            │                        │
   │── ③ 接続時に証明書を提示 ────────────────────────────▶│
   │                                                     │
   │                            （CA公開鍵で検証）         │
```

### サーバ証明書の種類

認証レベルに応じて3段階に分かれる。

| 種類 | 証明する内容 |
|------|------------|
| **DV証明書**（Domain Validation） | **ドメインの所有・管理**を証明する。最も安価で発行が速い。 |
| **OV証明書**（Organization Validation） | DV の内容に加え、**組織が実在する**ことを証明する。 |
| **EV証明書**（Extended Validation） | DV / OV の内容に加え、**業務実態**まで厳格に審査する。最も信頼性が高い。 |

### クライアント証明書

サーバではなく**クライアント側を認証する**ための証明書。VPN・ゼロトラストネットワーク・社内システムへのアクセス制御などで利用される。

## 失効した証明書の管理

何らかの理由（鍵漏洩・組織変更など）で**信頼できなくなった証明書を失効**させる仕組み。

### CRL（Certificate Revocation List、証明書失効リスト）

CAが管理する、**失効した証明書のリスト**。

- サイズが大きく、リアルタイム性に乏しい（更新間隔が長く最新化されにくい）。

### OCSP（Online Certificate Status Protocol）

証明書の失効状況を**プロトコル経由でオンラインに問い合わせる**仕組み。CRLの欠点を補い、リアルタイムに失効状態を確認できる。
